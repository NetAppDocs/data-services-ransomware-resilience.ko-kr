= NetApp Ransomware Resilience에서 복구 상태 검토
:hardbreaks:
:allow-uri-read: 
:icons: font
:imagesdir: ./media/


[role="lead"]
워크로드가 "복원 필요"로 표시된 후 NetApp Ransomware Resilience 실제 복구 지점(RPA)을 권장하고 충돌 방지 복구를 위한 워크플로를 조정합니다.

* VM이 NetApp Backup and Recovery 또는 Ransomware Resilience로 관리되는 경우, Ransomware Resilience는 시스템 충돌 시에도 동일한 시점에 볼륨에 있던 모든 데이터를 복원하는 충돌 일관성 복원을 수행합니다.
+
모든 볼륨, 특정 볼륨 또는 특정 파일을 선택하여 작업 부하를 복원할 수 있습니다.




TIP: 작업 부하 복구는 실행 중인 작업 부하에 영향을 줄 수 있습니다.  적절한 이해 관계자와 함께 복구 프로세스를 조정해야 합니다.

작업 부하에는 다음 복원 상태 중 하나가 있을 수 있습니다.

* *복구 필요*: 작업 부하를 복구해야 합니다.
* *진행 중*: 현재 복구 작업이 진행 중입니다.
* *복구됨*: 작업 부하가 복구되었습니다.
* *실패*: 작업 부하 복원 프로세스를 완료할 수 없습니다.




== 복구할 준비가 된 작업 부하 보기

"복원 필요" 복구 상태에 있는 워크로드를 검토합니다.

.단계
. 기본 Ransomware Resilience 대시보드의 알림 창에서 "복원 필요" 총계를 검토하고 *모두 보기*를 선택합니다.
+
또는 사이드바에서 *Recovery*를 선택하십시오.

. *복구* 페이지에서 작업 부하 정보를 검토하세요.
+
