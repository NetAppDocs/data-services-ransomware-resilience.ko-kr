= NetApp Ransomware Resilience 보고서 다운로드
:hardbreaks:
:allow-uri-read: 
:icons: font
:imagesdir: ./media/


[role="lead"]
NetApp Ransomware Resilience는 지원되는 볼륨과 지원되지 않는 볼륨, 공격 대비 훈련, 보호, 알림, 복구의 세부 정보를 보여주는 JSON 형식의 보고서를 제공하여 랜섬웨어 이벤트와 대비 상태를 오프라인에서 평가하고 검토할 수 있도록 합니다.


TIP: 파일을 다운로드하기 전에 대시보드를 새로 고쳐 보고서에 최신 데이터가 반영되도록 하세요.

*필수 콘솔 역할* 이 작업을 수행하려면 조직 관리자, 폴더 또는 프로젝트 관리자, 랜섬웨어 복원력 관리자 또는 랜섬웨어 복원력 뷰어 역할이 필요합니다. link:https://docs.netapp.com/us-en/console-setup-admin/reference-iam-ransomware-roles.html["NetApp Console 의 랜섬웨어 복원력 역할에 대해 알아보세요"^] .

*어떤 보고서를 다운로드할 수 있나요?*

* **요약**: 지원되는 워크로드와 지원되지 않는 워크로드 목록, 사이버 복원력 강화를 위한 권장 조치, 랜섬웨어 복원력 대시보드에 포함된 정보를 제공합니다.
* *보호*: 보호 대상 및 위험에 처한 워크로드의 총 개수를 포함하여 모든 워크로드의 상태 및 세부 정보를 제공합니다.
* *알림*: 모든 알림의 상태와 세부 정보, 총 알림 수 및 자동 스냅샷이 포함됩니다.
* *복구*: 복원이 필요한 모든 작업 부하의 상태와 세부 정보가 포함됩니다. 여기에는 "복원 필요", "진행 중", "복원 실패" 및 "복원 성공"으로 표시된 작업 부하의 총 수가 포함됩니다.
* *대비 훈련*: 개선이 필요한 영역을 강조하기 위해 모의 공격 결과를 포함합니다.
* *Clean restore*: Clean restore를 통해 복원된 워크로드에 대한 세부 정보가 포함됩니다.
+
정상 복원 보고서를 생성하려면 활성 격리 복구 환경(IRE)이 있어야 합니다. IRE가 삭제된 후에는 정상 복원 보고서를 사용할 수 없습니다.



.단계
. 콘솔 왼쪽 탐색에서 *보호* > *랜섬웨어 복원력*을 선택합니다.
+
image:screen-dashboard.png["대시보드 페이지"]

. 대시보드 또는 다른 페이지에서 오른쪽 상단의 *새로 고침* image:button-refresh.png["새로 고침 옵션"] 옵션을 선택하여 보고서에 표시될 데이터를 새로 고칩니다.
. 다음 중 하나를 수행하세요.
+
** 해당 페이지에서 *다운로드*를 선택하세요image:button-download.png["다운로드 옵션"] 옵션.
** NetApp Ransomware Resilience 메뉴에서 *보고서*를 선택합니다.


. *보고서* 옵션을 선택한 경우 미리 구성된 파일 이름 중 하나를 선택하고 *다운로드*를 선택합니다.
+
