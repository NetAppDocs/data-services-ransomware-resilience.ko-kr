= NetApp Ransomware Resilience에서 클린 복원을 통해 워크로드 복구
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
NetApp Ransomware Resilience를 사용하면 암호화 기반 랜섬웨어 공격 후 클린 복원을 통해 단계별 복구를 수행할 수 있습니다. 클린 복원은 데이터 손실을 최소화하고 워크로드를 최대한 빠른 시간 내에 온라인 상태로 복귀시킬 수 있도록 최적화된 복구 경로를 식별합니다.

클린 복원 옵션은 xref:task-custom-restore.html[사용자 지정 복원 옵션]에 대한 대안입니다.



== 시작하기 전에

클린 복원을 수행하려면 먼저 link:task-clean-restore.html["격리된 복구 환경"^]을(를) 구성해야 합니다.

*필수 콘솔 역할* 이 작업을 수행하려면 조직 관리자, 폴더 또는 프로젝트 관리자, 랜섬웨어 복원력 관리자 역할이 필요합니다. link:https://docs.netapp.com/us-en/console-setup-admin/reference-iam-ransomware-roles.html["NetApp Console 의 랜섬웨어 복원력 역할에 대해 알아보세요"^] .



== 고려 사항

* 암호화 기반 랜섬웨어 공격의 경우에만 클린 복원을 수행할 수 있습니다.
* 격리된 복구 환경에 새로운 작업을 처리할 용량이 없는 경우, 용량이 확보될 때까지 대기열에 추가됩니다.
+
** Ransomware Resilience 복구 대시보드에서 언제든지 활성 및 대기 중인 클린 복원 작업의 상태를 모니터링할 수 있습니다.


* 클린 복원을 시작하면 원래 볼륨이 마운트 해제되어 IO 액세스가 중단될 수 있습니다.




== 클린 복원 수행

. **Recovery** 섹션에서 복원할 워크로드를 선택합니다. **Restore**를 선택합니다.
. Restore Type 옵션에서 **Clean restore**를 선택한 다음 **Next**를 선택합니다.
. 사용할 격리된 복구 환경을 선택한 다음 **Next**를 클릭하십시오.
. 격리된 복구 환경을 검토하여 위치가 올바른지 확인합니다. **Restore**를 선택합니다.
. NetApp Ransomware Resilience가 필요한 설정을 수행합니다. 설정이 성공적으로 완료되면 NetApp Ransomware Resilience에 **Setup complete**라는 메시지가 표시됩니다. 분석을 진행하려면 **Next**를 선택하십시오.
. 설정이 완료되면 **Run analysis**를 선택합니다. Ransomware Resilience는 랜섬웨어 이벤트 이전 7일 동안 사용 가능한 모든 복원 지점을 분석하여 복원 지점의 날짜와 유형을 표시합니다.
+
image:screenshot-clean-room-analysis.png["클린 복원 분석 단계의 스크린샷."]

. 분석이 완료되면 **Next**를 선택하여 복구를 계획하십시오. Ransomware Resilience는 **Least data loss**와 **Latest unimpacted restore point** 두 가지 옵션을 제공합니다. 두 옵션 중 하나를 선택하고 선택적으로 각 복원 지점의 개별 파일 데이터를 확인하십시오.
+
세분화된 파일 이벤트 데이터를 보려면 파일을 선택하여 암호화 상태를 분석하고, 파일이 생성, 수정 또는 삭제된 시점과 각 작업에 해당하는 복원 지점을 확인하십시오.

+
image:clean-restore-plan.png["클린 복원 계획 단계의 스크린샷."]

. 복원 지점을 선택한 후 **다음**을 선택하여 파일 정리를 시작하세요.
. Ransomware Resilience가 워크로드 정리를 시작합니다.
+
정리가 완료되면 **Start recovery**를 선택하여 복구를 시작하십시오.

. 원본 워크로드를 저장할지 여부를 선택합니다. 원본 워크로드를 저장하지 않으려면 **아니요, 원본 워크로드를 교체합니다**를 선택합니다. 저장하려면 **예, 원본 워크로드를 저장합니다**를 선택한 다음 워크로드의 새 이름을 입력합니다.
. **Start recovery**를 선택하여 복구를 시작합니다.
. 복구가 완료되면 **Next**를 선택하여 최종 단계로 이동하십시오.
. 리소스를 해제하고 클린룸을 종료하려면 **Release resources and end**를 선택하십시오. 리소스를 해제할지 확인하려면 **End**를 선택하십시오.
